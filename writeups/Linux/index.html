every DevOps engineer will have to be proficient in linux to be better at his work. Linux is an operating system which is used mostly by security proffesional due to it customizabilty, fast usage and it being open sourced. without further ado let's get into how linux is being used in the field.

To create a folder or directory you use the mkdir command

Use mkdir new_folder to create a directory called "new_folder". NB:linux is case sensitive so "new" and "New" are not the same.
use the ls command to list what item inside a directory. it will just list the items but if you want to see all items plus hidden ones use ls -a to list all itemsincluding hidden ones.E.g; ls -a new_folder
Use the command ls -la to list everything inside of a directoryin a long list manner. E.g; ls -la new_folder

cat is used to read out files or output files to the screen.
touch is used to create empty files. eg. touch new_file
echo is used to output strings onto the screen

FILE PERMISSIONS

In linux there are different kind of file permissions since it is a multi-user operating system. every user is assigned it's own permissions. userA cant have access to userB without the proper permissions. when you list a directory with ls -la therefore making it list out all hidden items and normal items with their permissions, you will see something like drwxrwxrwx. now it has a meaning.

  1. dstands for directory. that is if that particular item is a directory, if its not it will be replaced with a dash
  2. rstand for read. that is if the user has read permissions on the file
  3. wstands for write. that is if the user has write permissions on the file
  4. xstands for executable. that is if the user can execute a particular file or script.
now that we know what it means lets go back and look at the permissions again, you will find out that the rwx appears thrice. the reason is every user has a group they belong to. so the first rwx of the permission belong to the user, the next one belongs to the group the user is in. and the last one belongs to other categories.

CHANGING PERMISSIONS

you might get to a time that you will want to change permissions of a user on a particular file, Linux gives us the opportunity to change permissions. first of all, to change permission you must be root or a sudo user. a sudo user is a user that can temporarily Use the root for sometime. The command to change permission is chmod. to remove the read and write permissio from the group, one can type sudo chmod g-rw new_fileyou will get the end result like rwx--x-rwx. This will remove the permission of the group and replace it with dashes.

CHANGING GROUP

To change the group a file belongs to, you will need the chgrp command. it changes the group a file belongs to, thereby making sure users in that group get access to it. all youhave to do is type sudo chgrp "group_name" file_name eg. sudo chgrp triddles new_file this changes whatever group there to triddles group.

CHANGING OWNERSHIP

to change the ownwership of a file from userA to another user, we use the chown command. we type out sudo chown old_owner:new_owner

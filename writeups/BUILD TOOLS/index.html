BUILD JS APPLICATION

If you're building an artifact with Javascript, it doesn't have a special artifact type. so we make it into a Zip or TAR file. We use npm and yarn. npm is widely used and popular. They uses package.json file to manage dependencies. npm and yarn are package managers NOT build tools. They install dependencies, but not used for transpiling Javascript code. When you try the npm, it downloads and install dependencies before the application can run. If you need some dependencies for your application, you can go to https://www.npmjs.com and search for it. npm uses different commands, some of them are :

  • npm start - start the app
  • npm stop - stop the app
  • npm test - run the tests
  • npm publish - publish the artifact

What does the zip/tar file include

It includes :

  • application code, but not the dependencies.
  • This means, you must install dependencies first before application will run
  • unzip TAR/ZIP
  • run the app
  • you need to copy artifact and package.json file to server
  • To create an artifact out of our node json file you can use npm pack

PACKAGE FRONTEND CODE

Now all the previous JS was for the backend code. to package the frontend, there are different ways;

  1. Package frontend and backend code separately, that is each code will have their own json files.
  2. A common package for both.
The frontend code needs to be transpiled otherwise the browser cant render it. browsers mostly don't support latest JS versions or other fancy code decorations so we need to transpile that to standard JS syntax. It also needs to compress so its not too large for the browser to download, hence loading faster for the end-user. you can use Webpack to build code, transpile, minify and compress the code. it makes it more smaller for browsers to be able to work with.

Common concepts and differences

These concepts are similar to other languages, in python, we use pip package manager. There are some patterns in these tools too. They all have a dependency file. they have repository for the dependencies. they're all command line tool. they all act like package managers.

Publish an artifact.

Build tools have tools for publishing the created artifacts, saves the artifacts to be hosted on the server. You can download it anytime and anywhere you want. We publish and build a docker image instead of having all that different artifact types. we build those docker images from the application. so there is no need for a repository for each of the file type. JUst one docker image. There is no need to install dependencies on the server.you execute everything inside image.

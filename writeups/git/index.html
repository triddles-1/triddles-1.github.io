.gitignore

.gitignore is not a command but rather a text file that contains some informatiion, telling git to ignore some files.

  • Add .gitignore file to your project's root directory
  • It is used to exclude certain files or folders from being tracked by git
  • folders like build, .ideas and node_modules should not be tracked
  • list all folders and files you dont want to be tracked inside .gitignore file
  • If there is a tracking on folders because you have already committed some of them, you can use git rm -r --cached file/folder
  • after that, check git status, commit and push

GIT STASH

By typing git stash it saves the curent changes for later. now you can checkout to anoter branch without any error message. To get back to your changes use git stash pop .
Another case use is when we ake changes and there's a break in the app. we can use stashing to hide our changes temporarily to test if app works without our code changes.

Undoing commits

when you commit locally without pushing, then you realize that you made a mistake or tere's still some changes to be made. you use "reset --hard HEAD~1" for the last commit. if you want the last but one commit you change it to HEAD~2.
you can use git reset --soft HEAD~1 to keep changes but still be allowed to make changes to the commit. soft is also the default option, so even if you remove "-soft" from the command it still works. use git commit -amend to amend the commit messages and also doesn't change to the last commit.
when you push you cant revert with git reset as you did locally, but you can do that with git push --force Do not do this in master or develop branch or a branch where a lot of people work together

git revert commit_hash will revert to a commit assuming there's a lot of commits. it doesn't override but creates a new commit that contains the old changes.

DATABASE FOR DEVOPS

By Triddles

Databases in software developments

Developers need databases for local developments. they need them to test how the application really works. To use the database in the development phase, there are three options you can use.

  • option 1: database hosted locally
    • each developer install the database locally, eg, Mysql
    each developer has his own dataset that he is using.
    • Pros:
      • you can't mess with someone else's test database data
    • Cons:
      • you have to start from an empty database
      you need realistic test data
      install and setup database for every developer
  • option 2: DAtabase hosted remotely
    • pros:
      • no needd for local installation
      start coding right away
      test data is available right away.
    • cons:
      • cant play around without affecting others.
  • option 3: Database have both local and remote

How does the app talks to the database

  • In your application code you configure the database connectivity.
  • Each programming language has a library/modules for database connection
  • You have to tell the library, which database to talk to and how to AUTHENTICATE with that database.
    • you also need credential and the database endpoint..

Do not hardcode the database endpoint and credentials. Define them in one place as enviroment variables so you can confi from outside. To change database, we just change it from the configuration files.

Depending on environment(dev,test,prod) it all connects to different databses. Each has a different endpoint and credentials with production being more secured.
In the config files;

  • Define variable in code
  • Set endpoint and credentials from outside each environmnet
  • Pass environment variables on app start up through command line or configuration in code editor (impractical)
  • Before Appis deployedd, we need to install and configure database
  • we can set up database on the same server as the application or we use a separate server for database.
  • Data is important so we backup, replicate and make sure it performs under high loads/traffic
so a devops engineer should know how to configure a database, how to setup database, manage databse and restore database.

DATABASE TYPES

  1. Key Value Database:
    • any Key value is unique andpoints to a value
    • you can read data using key
    • you have no joints
    • They are very fast because they store data in memory
    • You cant store data
    • it is not fit as a primary database
    • it is used for caching
    • Twitter and snappchat uses these for real-time data delivery
    example of key value databases include:
    • Redis
    • Memcached
    • etcd from Kubernetes
  2. Wide Column Database:
    • it is used to store more complex value data
    • one key divides the value into multiple columns
    • it is
    • very scalable
    • Queries similar to SQL
    • no joints
    • It is best for TIMED-SERIES
    • used for keeping records of IoT devices
    • used for keeping historical records of an application
    example of wide column database are:
    • cassandra
    • Apache Hbase
  3. Documented oriented Database:
    • it's schema-less
    • documents are grouped in collection
    • collections can be organized into relational hierachy
    • no joins
    • faster to read
    Best used for:
    • Mobile apps
    • Game apps
    • Content management Services (CMS)
    examples are :
    • MongoDB
    • DynamoDB
    • CouchDb
  4. relational Database:
    • it is the most used databse type
    • the data are structured
    • Strict schema and data types needs to be created first
    • It uses Structured Query Language (SQL) for reading and writing
    • Data is stored in tables
    • it contains joins
    examples are:
    • Mysql
    • Postgresql
  5. Graph Database
    • Directly connects entities
    • Edges are the relationship
    • Easier to Query
    Best used for:
    • detecting patterns in application data
    • detecting relation between records
    • used by recommendation engines
  6. Search Database:
    • Search databse through massive data entries
    • Full text search in efficient and fast way
    • similar to document-oriented database
    exampl are;
    • elastic search
    • Solr

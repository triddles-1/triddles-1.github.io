AWS OVERVIEW

AWS stands for Amazon Web Service. it is a service that offers service such that you can host your information, application on their service for a fee. It is one of the most used web services. they have workstations over the world that host their servers.

Identity access management(IAM) is one key cocept in AWS. you need IAM to manage wat and which resource a user is entitled to. IAM principals must be authenticated to send requests (with a few exceptions).
A principal is a person or an applicaion that can make a request for an action or operation on an AWS resource. principals comprises of User, Role, federated user and applicaion.
Policies control access to resources, it determines whether to authorize the request. everything in AWs is an API call. If you try to get informatiionon an EC2 instance, you will run a "Runinstance" api call. If you want to get informatiion on an S3 bucket, you will run the "GetBucket" call. If you want to create an IAM user you will use use the CreateUser api call.

SETUP INDIVIDUAL USER ACCOUNT

there are two types of AWS users. that is;

  1. Root user: the user is creaated with the email address used when creating an aws account in the first place. This user has full access to aws resources. It is a very poerful user, so it's best practice not to use it often.
  2. IAM user: A user created by the root user. it has a name and an alias or AWS Account ID. Its action is determined by the IAM permission policy.

CREATING AN IAM USER

  1. Search for IAM in the AWS console and select it
  2. click on add users
  3. GIve the user a name
  4. Provide access to AWS management console(optional)
  5. Select i want to create IAM user
  6. Set password, it can be auto-generated or custom
  7. you can choose to let user change passwords at login or not
  8. click on next
  9. Add permissions to user(optional)
  10. Click on next
  11. Now user will be created, you can login with the credentials you just created.

One of the best ways to assign permissions to users is through groups. To do that;
  1. Select user groups in IAM console
  2. click on create groups
  3. give the group a nam. eg. Admin
  4. let's give the admin group full permissions
  5. you will find a checkbox with the name "AdminstratorAccess", check it
  6. select group
    7. when you click on the plus sign next to AdminstratorAccess you will see the policy written in json. Now you're set to add users. you can go into the group and add users.

AMAZON ELASTIC COMPUTE CLOUD (EC2)

EC2 is a way we run virtual services in the cloud. EC2 host servers in the cloud for us. It uses server virtuallization. you you launch an instance, you have to choose and instance type. that is, you select your CPU, memory, storage and networking. It determines the hardware performances and the cost.
To launch an EC2 instance;

  • Go to the AWS console and select EC2
  • Select instance type
  • Select the Amazon Machine Image(AMI) that you want. it selects the operating System(OS) and configuration.

BENEFITS OF EC2

  1. Elastic Computing: you can easily launch hundreds of thousand of EC2 instances within minutes
  2. Complete Control: you control the EC2 instance with full root/administrative access
  3. Flexible: choose your choice of instance types, operating systems and software packages.
  4. Reliable: EC2 offers a high levels of availability and instances can be rapidly commissioned and replaced.
  5. Secure: Fully integrated with AmAzon VPC and security features.
  6. Inexpensive: Low cost, pay what you use.

LAUNCHING EC2 INSTANCES(LINUX & WINDOWS)

launching a linux EC2 instance
  1. Select launch instance in the EC2 page
  2. Give it a name
  3. Choose OS image (Amazon Machine Image). The default is Amazon Linux.
  4. Choose instance type(you will see pricing per hour)
  5. Create a key pair. it is a private and public key.
  6. Configure your networking
    • Leave VPC and sunet, unless you have a vpc and subnet you want to change to.
    • Public IP is allowed
    • Create a new security group and description.
    • Set up rules (allow SSH for now)
    • Source type should be anywhere
  7. launch instance
WINDOWS
  1. Click on launce instance
  2. Select windows as the OS image
  3. Choose instance type
  4. Choose key pair. that is the one we already created. if not create one.
  5. Configure network settings. leave everything except we configure existing group by selecting existing one
  6. Launch Instance
